Trigger panel

This panel holds the list of triggers generated by detections jobs.



The user has to review each trigger reported before committing it into the SeisComP3 system.

At startup, the application will load triggers that are accepted (but not committed yet) and the ones that are waiting for user revision.


Various buttons offer the user simple actions like :

  • Accepting a trigger, mandatory in order to commit an origin,
  • Rejecting a trigger,
  • Reseting a wrongfully rejected trigger,
  • Committing accepted triggers,
  • Deleting a trigger,
  • Cleaning committed triggers automatically.


Note

Once a trigger deleted, the only way to retreive it is to reset then re-run the job.
