SeismicDataPlayback installation

Requirements

The hardware requirements for performing dispatch and detection depend on the volume of data to be processed, and the amount of time the user has to perform the analysis.

As a nominal configuration, SDP has been tested to opperate 3 slots (threads) inside a virtual machine running :

  • 1 CPU
  • 4GB RAM
  • HDD 40GB
A more athletic configuration would certainly help process scripts faster.



O.S.

SDP can be ran and has been tested on various operating systems like Linux or Unix.

Prior to installation, the user must install the following libraries :

  • python-obspy
  • gcc/g++
  • cmake
  • qt4-dev
  • rdseed
  • msmod
  • msrouter

msrouter, msmod are compiled by the installer, but the user can specify its own by editing the configuration file.
rdseed binary which is mandatory for using binary dataless for populating the station inventory should be registered in user's $path, or specified in configuration file.



Installation procedure

The next steps describe the installation of SeismicDataPlayback from the source code :

  • go inside sdp folder
    cd ~/install-sdp/sdp/
  • create build subfolder
    mkdir build
  • go inside build folder
    cd build/
  • start compiling
    cmake .. && make
  • install the application
    make install
